Week 05

Notes:

  • Stack traces help us find the source of the error
  • If there are warnings in our code, we must correct them because even if the code continues to work, this can cause problems in its operation
  • Writing code in strict mode helps us with clarity and speed
  • If we applied strict mode on a function it will be applied inside the function
  • Breakpoints help us run a step-by-step run of our applications looking for errors
  • The alert() method allows us to set breakpoints at which we can check the state of our variables
  • Browsers also allow us to set breakpoints to analyze our applications
  • With the 'debugger' command we can set breakpoints to analyze our code
  • When we use the 'debugger' command we have to make sure to delete all these commands
  • We can create errors as objects and pass them as a parameter a text that indicates the error
  • Try, Catch, and finally are used to handle errors and allow us to clearly detect the points where we think errors may arise.
