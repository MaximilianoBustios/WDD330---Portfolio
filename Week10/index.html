Week 10

Notes:

  • Built-in validation is called on everything related to validation in the HTML document
  • With the pattern attribute, we set regular expressions as is normally done in Javascript
  • With the pseudo-class :invalid we can highlight field errors so that the user can easily see them.
  • The textarea element does not support the pattern attribute.
  • The Constraint Validation API enables checking values ​​that users have entered into form controls, before submitting the values ​​to the server.
  • The Constraint Validation API works in conjunction with the attributes we set on the client-side to validate input.
  • When we decide not to use the built-in API, the changes we have to make to javascript are quite large. It is possible but requires more code.
  • We must always be permissive with the input formats, if we can correct something we can do it, as long as they are not passwords.
  • When there is an error, we must focus on that error to easily show the user what to correct.
  • Fetch gives us a more flexible way of working with the server.
  • The fetch() method can optionally accept a second parameter, an init object that allows controlling a number of different settings.
  • A fetch() promise will reject with a TypeError when a network error is encountered or CORS is misconfigured on the server-side
  • When fetch() promises are resolved, different responses can be received, the most common are: Response.status, Response.statusText, and Response
